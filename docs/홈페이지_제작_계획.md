## 1. 프로젝트 한 줄 요약

부산의 한·일 일본어 스터디 모임 **소코소코(Sokosoko)** 를, 일본인 관광객 및 한국 거주 희망 일본인이 **유기적으로 발견하고 신뢰할 수 있는 단일 통합 플랫폼**으로 만들기 위한 다국어 홈페이지 구축.

- **브랜드명(JP)**: 「**釜山暮らし。いもじょも**」 (와이어프레임에서 확정)
- **사이트 성격**: 「유학·생활 정보 블로그」를 미끼로 한 「소코소코 모임 유입 통로」

회의에서 명확히 합의된 방향:

- 회원 수를 늘리기 위함이 **아님** (이미 약 300명, 연내 마감 예정)
- **일본인 유입 채널 부재**가 핵심 문제 (네이버·인스타로는 한계)
- 단순 홍보가 아니라 **"정보를 미끼(먹이)로 자연 유입"** 을 만든다
- 장기적으로 **관광객 대상 체험 패키지(스터디 + 뒤풀이 + 친구 만들기)** 로 확장

### Phase 1 — 기본 홈페이지 오픈 (MVP)

회의 합의: "처음에는 심플하게, 기능 적게."

#### 1-1. 정적 페이지

- **메인 (Top page)**:
  - 상단: 일러스트 메인 비주얼 (광안대교 캐릭터, 와이어프레임 그대로)
  - 브랜드 카피: 「釜山暮らし。いもじょも」
  - **3-pillar 카테고리 카드** (暮らす / 学ぶ / 旅する) ← Shizuoka 패턴
  - **New?** 신규 글 3개 카드
  - **소코소코 모임 안내 CTA** (와이어프레임 보강 항목, 정보 사이트로만 보이지 않게)
  - SNS 링크 (Instagram / X / LINE / note)
- **About 페이지**: 모임 철학("건전한 관계 지향"), 활동 사진, 운영진
- **참여 안내**: 요일별 반(초·중·고급) 안내, 대기 신청 폼
- **Contact**: 이메일 / 인스타 / 네이버 카페 / X 링크

#### 1-2. 정보 블로그 (Info Blog)

- 카테고리: 와이어프레임 3-pillar 그대로 (暮らす / 学ぶ / 旅する)
- 작성자 프로필 한 줄 (예: "부산 거주 N년 / ◯◯대학원 재학") → 신뢰도 부여
- 작성 주기: 주 1회 1건 이상 (운영진 10명 로테이션)
- AI 초안 작성 도구는 보조로만 사용

#### 1-3. 다국어 (한국어 / 일본어)

- IP 기반 자동 + **헤더 우상단 JP|KR 토글** (LOG IN 좌측, 와이어프레임 보강)
- 콘텐츠는 처음부터 양 언어 동시 작성 원칙

#### 1-4. SNS 자동 동기화

- 블로그 발행 시 X·Instagram 자동 포스팅 (API 연동)
- 공유 버튼 (X, Instagram, LINE, note) 모든 글에 노출
- ※ 인스타 그래프 API 제약 사항 사전 확인 필수

#### 1-5. SEO 기반

- Google 우선 (sitemap.xml, robots.txt, structured data)
- 핵심 키워드: `釜山 日本語`, `釜山 留学`, `釜山 ワーホリ`, `釜山 暮らし`, `한국 유학`
- 글 작성 가이드라인: 제목 길이, 본문 분량(최소 800자), 이미지 alt, 내부 링크
- Google Search Console / Analytics 초기부터 연동

#### 1-6. 인프라

- 외부 호스팅 (예: AWS Lightsail / Cafe24 / Vultr Tokyo 등 후보 비교)
- HTTPS(SSL) 필수
- 백업: 미니 PC 보조 백업 (이미 보유)

**Phase 1 종료 조건:** 운영진 20명 내부 베타 테스트 → 글 작성·다국어·SNS 동기화·검색 노출 확인 → 정식 오픈.

### Phase 2 — 커뮤니티 기능

#### 2-1. 회원 등급 체계

| 등급                                  | 권한                | 시각 표시    |

- **정보 게시판**: 인증 회원만 작성 / 누구나 열람

- **자유 게시판**: 정식 회원 작성, 후기·잡담
- **활동 게시판**: 운영진 작성, 모임 소식·이벤트

- 우측 사이드바 **Rule 박스** — Q&A 작성 규칙
- 우측 사이드바 **Question Tips 박스** — 질문 잘 쓰는 법

#### 2-4. 보안·법무 (필수)

- 개인정보 암호화 저장 (비밀번호 bcrypt 등)
- 개인정보처리방침 / 이용약관 게시 + 가입 시 동의 절차
- AI 기반 1차 스팸·욕설 필터 + 신고 기능
- 관리자 모니터링 대시보드 (회원 강퇴, 글 삭제)

### 5.5 카드 컴포넌트 사양


**카드 동작**

- 카드 전체 클릭 → 글 상세 페이지
- 카테고리 배지 클릭 → 해당 카테고리 목록
- 태그 클릭 → 해당 태그 목록
- 작성자 마크 클릭 → 작성자 글 모음
- 호버: 약한 그림자 + 살짝 들림 (lift)
- 행 내 카드 높이 균등 (felissimo 패턴)

## 6. 기술 스택 권장안

| 영역         | 1안 (권장)                                                    | 2안                    |
| ------------ | ------------------------------------------------------------- | ---------------------- |
| 프레임워크   | **Next.js** (App Router, i18n 내장)                           | Astro + CMS            |
| CMS / 콘텐츠 | **Headless (Sanity / Strapi)** — 운영진이 비개발자라 GUI 작성 | Markdown 기반          |
| DB           | PostgreSQL (호스팅 업체 관리형)                               | MySQL                  |
| 인증         | NextAuth + 이메일 인증                                        | Supabase Auth          |
| 호스팅       | Vercel + 관리형 DB                                            | AWS Lightsail / Cafe24 |
| SNS 연동     | X API v2, Instagram Graph API                                 | Zapier / Make 우회     |
| 검색·분석    | Google Search Console + GA4                                   | —                      |

이전에 시도했던 **Wix**는 게시판·회원 등급·SNS 자동 연동 측면에서 한계가 있어 재사용 비권장.

### 6.1 DB 필드 요건 (v1.1 추가)

와이어프레임 + 카드 컴포넌트로부터 도출:

**posts 테이블 필수 필드**

- `id`, `title_jp`, `title_kr`, `body_jp`, `body_kr`
- `category` (enum: kurasu / manabu / tabisuru / about / etc)
- `subcategory` (예: visa, arc, food, ...)
- `tags` (배열)
- `thumbnail_url` (16:9 권장)
- `author_id` → users.id
- `published_at` (NEW 라벨 판정용 — 7일 이내)
- `view_count` (Top topics TOP 5 정렬용)
- `like_count`
- `is_featured` (boolean — Featured/Recommended Picks 큐레이션용)
- `is_recommended` (boolean — 편집자 추천 슬롯)
- `created_at`, `updated_at`

**users 테이블 필수 필드**

- `id`, `email`, `password_hash`
- `nickname_jp`, `nickname_kr`
- `grade` (enum: guest / member / verified / certified / admin)
- `profile_one_liner` (예: "부산 거주 3년 / ◯◯대학원 재학")
- `avatar_url`
- `agreed_to_terms_at`, `agreed_to_privacy_at` (법적 동의 타임스탬프)

## 7. UI/UX 원칙

1. **심플 제일주의** — "조잡하면 사람들은 눈을 어디 둘지 모른다" (대표 발언)
2. **선택과 집중** — 부산나비처럼 정보 백화점 X. 소코소코 + 정보 블로그 두 축만.
3. **일본인 가독성** — 한국 사이트 특유의 화려함 회피
4. **신뢰 기호의 시각화** — 왕관/사과 마크, 작성자 프로필 한 줄
5. **모바일 최적화** — 일본인 이용자 모바일 비율 높음
6. **3초 룰** — 첫 화면에서 "여기는 무엇을 하는 곳"이 즉시 파악되어야 함
7. **3-pillar 일관성** — 헤더·메인·블로그·푸터 모두 暮らす/学ぶ/旅する 동일 아이콘·라벨
8. **카테고리 발견성 우선** — 시간순보다 관심사 기반 (felissimo 패턴)

## 8. 콘텐츠 전략

### 초기 시드 콘텐츠 (오픈 전 준비)

운영진 10명 × 1편 = **10편 사전 작성** 후 오픈.

**카테고리 분배 (와이어프레임 사이트맵 기반 재배치)**

- **学ぶ 4편**: 어학당 입학 / 대학 지원 / VISA 신청 / ARC·연장
- **暮らす 4편**: 방 구하기 / 쓰레기 분리수거 / 운전면허 / 은행 계좌
- **旅する 1편**: 부산 지하철·KTX 가이드
- **About / 소코소코 소개 1편**

### 운영 룰

- 주 최소 1편 발행 → 발행 즉시 X·인스타·LINE·note 자동 송출
- 6개월 이상 지난 글은 "업데이트" 또는 "아카이브" 표기
- 키워드 가이드라인을 별도 1페이지로 운영진에게 배포
- **Featured / Recommended Picks** 는 매주 대표 또는 케코가 1회 수동 큐레이션
