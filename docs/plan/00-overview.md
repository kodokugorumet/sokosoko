# 00. Overview — 한 줄 요약과 핵심 의사결정

## 1. 프로젝트 한 줄 요약

부산의 한·일 일본어 스터디 모임 **소코소코(Sokosoko)** 를, 일본인 관광객 및 한국 거주 희망 일본인이 **유기적으로 발견하고 신뢰할 수 있는 단일 통합 플랫폼**으로 만들기 위한 다국어 홈페이지 구축.

- **브랜드명 (JP)**: 「**釜山暮らし。いもじょも**」 (와이어프레임에서 확정)
- **브랜드명 (KR)**: 「**부산 살이. 이모저모**」
- **사이트 성격**: 「유학·생활 정보 블로그」를 미끼로 한 「소코소코 모임 유입 통로」

## 2. 회의에서 명확히 합의된 방향

- 회원 수를 늘리기 위함이 **아님** (이미 약 300명, 연내 마감 예정)
- **일본인 유입 채널 부재**가 핵심 문제 (네이버·인스타로는 한계)
- 단순 홍보가 아니라 **「정보를 미끼(먹이)로 자연 유입」** 을 만든다
- 장기적으로 **관광객 대상 체험 패키지(스터디 + 뒤풀이 + 친구 만들기)** 로 확장

## 3. 핵심 의사결정 (회의 + 와이어프레임 결과)

| 항목          | 결정 사항                                                | 근거                                            |
| ------------- | -------------------------------------------------------- | ----------------------------------------------- |
| 브랜드명      | 「釜山暮らし。いもじょも」 / 「부산 살이. 이모저모」     | 와이어프레임 메인 시안                          |
| 타겟          | 일본인 우선 (한국인 유입은 기존 채널 유지)               | 일본 통계청: 일본인 방문 도시 5위 부산          |
| 검색 채널     | **Google 우선**, 보조로 X(트위터)·note                   | 카렌 씨가 note 블로그 통해 소코소코 발견한 사례 |
| 확산 전략     | 홈페이지 1회 작성 → X·인스타 **자동 동기화**             | 수기 동기화는 의지 떨어져 지속 불가             |
| 카테고리 구조 | **3-pillar (暮らす / 学ぶ / 旅する) + Q&A + About**      | 와이어프레임 사이트맵 + Shizuoka 4-pillar 모델  |
| 회원제        | **회원가입 필수 + 등급제(왕관/사과 마크)**               | 1차 필터링 + 신뢰성 차등 부여                   |
| 게시판 구조   | 정보 / 자유 / Q&A / 활동 4분할, 작성 권한 차등           | 쓰레기 정보·홍보 글 차단                        |
| 디자인 톤     | **심플 + 일러스트 + 일본인 가독성**                      | 와이어프레임 메인 일러스트 (광안대교 캐릭터)    |
| 콘텐츠 공급   | 일본인 운영진 10명 → 주 1회 글 작성                      | 한 명 의존 시 번아웃 (이전 블로그 실패 경험)    |
| 인프라        | **Cloudflare Workers (OpenNext) + Sanity 호스팅 Studio** | 무료, 글로벌 edge, 상업적 사용 허용             |
| 회원가입 시   | 개인정보보호법 준수 + 약관·처리방침 작성                 | 법적 자문 필요 (AI 초안 + 검토)                 |

> **호스팅 결정 변경 이력**: 초기 계획(v1.1)에서는 Cloudflare **Pages** 였으나, Next.js 16 의 `@cloudflare/next-on-pages` 가 deprecated 되어 **Workers + `@opennextjs/cloudflare`** 로 변경. 임베디드 Sanity Studio 는 Worker 크기 3 MiB 제한을 초과하여 **Sanity 호스팅 (`*.sanity.studio`)** 으로 분리. 자세한 근거는 [`docs/adr/0002-hosting-cloudflare-workers.md`](../adr/0002-hosting-cloudflare-workers.md) 참조.
