# Runbook: 콘텐츠 작성 가이드 (운영진용)

> 운영진이 Sanity Studio 에서 글(post) · Q&A · 작가(author) · 카테고리(category) 를
> 작성·발행하는 절차. 이 사이트는 일본어/한국어 양언어를 한 문서 안에 같이 보관한다.
>
> 관련 NFR: NFR-MAINT-06 (스키마 = 단일 진실 공급원), NFR-I18N-01~04 (양언어)

## 0. Studio 접속

- URL: `https://sokosoko.vercel.app/studio`
- 로그인: 본인의 Sanity 계정 (운영자가 미리 invite). 처음 접속이면 관리자에게 초대 요청.
- 좌측 사이드바: **記事 / 글**, **質問 / 질문**, **著者 / 저자**, **カテゴリ / 카테고리**, **タグ / 태그** 5종.

## 1. 작가(Author) 등록 — 한 번만

처음 글을 쓰는 멤버는 본인 author 도큐먼트를 먼저 만든다. 이미 있다면 건너뛴다.

1. 사이드바 **著者 / 저자** → **+ Create**
2. 필드:
   - **名前 / 이름** — `ja` 와 `ko` 모두 입력 (예: `いもじゃも` / `이모자모`)
   - **Slug** — 자동 생성 후 `Generate` 클릭 (예: `imojamo`)
   - **Member tier** — 본인 등급 선택
     - `👑 Crown` — 운영진 / 정기 기고자
     - `🍎 Apple` — 멤버 기고자
     - `✏️ Contributor` — 일회성 기여자 (기본값)
   - **自己紹介 / 자기소개** — 한 문단 정도 (양언어, 선택)
   - **アバター / 아바타** — 정사각형 이미지 권장 (선택)
3. **Publish** (오른쪽 아래 초록 버튼)

## 2. 카테고리(Category) 확인

글은 반드시 카테고리에 묶이고, 카테고리는 3개의 pillar (`life` / `study` / `trip`) 중
하나에 속한다. 보통 운영자가 미리 만들어 두므로 새로 만들 일은 적다.

- 신규 카테고리가 필요하면 **カテゴリ / 카테고리** → **+ Create**
- 필드:
  - **Pillar** — `life` / `study` / `trip` 중 선택 (UI 의 라디오)
  - **Title** — `ja` / `ko`
  - **Slug** — `Generate`
  - **Description** — 짧게, 양언어
  - **Order** — 정수 (작을수록 먼저 노출)

## 3. 글(Post) 작성

### 3-1. 새 문서 만들기

1. **記事 / 글** → **+ Create**

### 3-2. 필수 필드

| 필드                | 입력 방식                                                                                  |
| ------------------- | ------------------------------------------------------------------------------------------ |
| **タイトル / 제목** | `ja` 와 `ko` 둘 다. 둘 다 채우는 것이 원칙                                                 |
| **Slug**            | `ja` 제목으로부터 자동 생성 → `Generate` 클릭. 나중에 URL 이 되므로 깔끔한 영문/숫자 권장. |
| **抜粋 / 요약**     | 카드/검색 결과/OG 에 노출되는 한두 문장. 양언어 모두.                                      |
| **カバー画像**      | 16:9 권장. 업로드 후 hotspot 을 인물/주피사체에 맞춰 드래그.                               |
| **카테고리**        | 2 단계의 카테고리 중 하나                                                                  |
| **저자**            | 본인 (1 단계의 author)                                                                     |
| **公開日 / 공개일** | 발행 시점. 미래 날짜로 두면 예약 발행 효과는 없고 그냥 그 날짜로 표시됨.                   |

### 3-3. 본문(body) — 양언어

- **本文（日本語）** 와 **본문 (한국어)** 두 필드가 분리되어 있다.
- **둘 다 채우는 것이 원칙**이지만, 한쪽만 있어도 발행 가능 — 사이트가 자동으로
  fallback 배너 ("이 글은 한국어 번역이 아직 준비되지 않아 일본어 원문을 보여드립니다") 를 띄운다.
- 본문 에디터 (Portable Text) 사용법:
  - 위쪽 툴바: **H1 / H2 / H3** 헤딩, **Bold** / **Italic**, 인용, 리스트, 링크
  - 줄바꿈은 Enter (단락) 또는 Shift+Enter (소프트)
  - 이미지: 본문 가운데 클릭 후 좌측 `+` 아이콘 → **Image** 선택 → 업로드
  - 링크: 텍스트 선택 → 툴바의 🔗 → URL 입력 → external 은 `Open in new tab` 체크 권장

### 3-4. 글 작성 시 체크리스트

- [ ] 제목 (양언어)
- [ ] Slug (Generate)
- [ ] 요약 (양언어, 1~2 문장)
- [ ] 커버 이미지 + hotspot
- [ ] 카테고리 + 저자 + 공개일
- [ ] 본문 일본어 (또는 한국어)
- [ ] 본문 한국어 (또는 일본어)
- [ ] 미리보기에서 H2 위계가 자연스러운지
- [ ] 외부 링크는 새 탭 열기

### 3-5. 발행

1. 화면 하단 **Publish** 클릭
2. 1~2 초 후 webhook 이 ISR 캐시를 무효화하면 즉시 사이트에 반영
3. `https://sokosoko.vercel.app/<pillar>/<slug>` 에서 새 글 확인 (예: `/life/hello-busan`)
4. 반영이 안 보이면 **Sanity Webhook 등록 여부**를 먼저 확인:
   [`docs/runbooks/sanity-webhook.md`](./sanity-webhook.md)

## 4. Q&A 작성

질문은 글보다 단순하다 — 카테고리 / 작가 / 커버 이미지 없음.

1. **質問 / 질문** → **+ Create**
2. 필드:
   - **質問 / 질문** — 한 문장으로, 양언어 (예: `釜山で家を探すコツは？` / `부산에서 집 구하는 팁은?`)
   - **Slug** — `Generate`
   - **分類 / 분류** — `life` / `study` / `trip` / `other` 중 하나 (라디오)
   - **注目 / 주요 질문** — 상단 고정하고 싶으면 ON
   - **受付日 / 접수일** — 질문 받은 날짜
3. **回答（日本語）** / **답변 (한국어)** — Portable Text. 한쪽만 있어도 OK (글과 동일하게 fallback).
4. **Publish**
5. `https://sokosoko.vercel.app/qa/<slug>` 에 즉시 노출.

## 5. 이미지 작성 규칙

- **포맷**: JPG 또는 PNG. SVG 는 본문 인라인 X (커버 X).
- **해상도**: 가로 1600px 이상 권장 (Sanity 가 자동 리사이즈/WebP 변환).
- **alt 텍스트**: 커버 이미지의 alt 필드는 양언어 입력 — 시각장애인 + SEO 양쪽에 영향.
- **저작권**: 본인 촬영 / 라이선스 명확한 무료 소스 (Unsplash, Pexels) 만. 출처 불명 사진 X.

## 6. 글쓰기 톤

- **1 인칭 OK**, 단정 어조 OK. 가이드북 X, 친구의 경험담 ○.
- **한쪽 언어로 먼저 쓰고** 운영진 검수 후 다른 언어로 옮기는 흐름 권장 — 양언어를
  동시에 쓰면 둘 다 어색해지기 쉽다.
- **외부 링크**: 한국 사이트 (네이버 지도, 카카오맵 등) 와 일본 사이트 (Google 등) 를
  맥락에 따라 골라 쓴다. 두 독자층 모두를 의식.
- **이름 표기**: 처음 등장 시 양언어 병기 (예: `광안리(クァンアンリ)`).
- **숫자 / 가격**: ₩ 와 ¥ 를 함께 적으면 친절 (환율 변동은 명시 안 해도 됨).

## 7. 자주 묻는 운영 질문

**Q. 한쪽 언어만 쓰고 발행해도 되나요?**
A. 가능하지만 사이트에 fallback 배너가 표시됩니다. 가능하면 같은 주에 다른 언어도 채워주세요.

**Q. 잘못된 글을 발행했어요. 어떻게 되돌리나요?**
A. Studio 에서 그 문서를 열고 우상단 `History` → 이전 버전 클릭 → `Restore`. 또는
직접 수정 후 다시 Publish (webhook 이 즉시 반영).

**Q. 글을 비공개로 내리고 싶어요.**
A. 문서를 열고 우상단 `⋮` → **Unpublish**. 사이트와 sitemap 에서 즉시 사라집니다.

**Q. Slug 를 바꾸고 싶어요.**
A. 가능은 하지만 기존 URL 의 SEO 가 사라지고 백링크가 깨집니다. 발행 후에는 가급적 변경 금지.

**Q. 글이 사이트에 안 보여요.**
A. 체크 순서:

1. 문서가 **Published** 인지 (Draft 상태가 아닌지) — 우상단 상태 표시 확인
2. **公開日** 가 과거 날짜인지
3. **slug** 가 비어있지 않은지
4. Webhook 이 등록되어 있는지 (운영자에게 문의)
